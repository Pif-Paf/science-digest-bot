## ✨ Новые возможности v2.0

### 1️⃣ **Поддержка Hugging Face** 
- Сбор трендовых моделей и датасетов
- Анализ по тому же принципу: суть, важность, реакция сообщества
- Интеграция с существующим анализом

### 2️⃣ **Структурированный вывод с рейтингом и тегами**
- **⭐ Рейтинг (1-10 баллов)** — для быстрой оценки важности
- **🏷️ Теги (#ML, #Security, #NLP и т.д.)** — для поиска и категоризации материалов
- Красивая разметка Markdown для читаемости

### 3️⃣ **Podcast-style аудио вместо просто чтения**
- **Вместо:** просто автоматическое чтение текста вслух
- **Теперь:** сценарий диалога между двумя ведущими (Алексей & Мария)
- Обсуждение ТОП-3 материалов, рассуждения, вывод
- Более естественное и интересное восприятие

### 4️⃣ **Метаданные и архив**
- Сохранение полной истории в JSON:
  - Полный текст дайджеста
  - Все собранные материалы с метаданными
  - Podcast-скрипт
  - Статистика источников
- Файл: `digest_YYYYMMDD.json`

## 🏗️ Где и как собираются статьи

### **Источник 1: arXiv** (`fetch_arxiv_papers`)
```
🔍 Поиск: cs.CR (Security), cs.LG (ML), cs.AI (AI)
⏱️ Фильтр: последние 2 дня
📊 Объём: до 6 статей
📦 Данные: title, authors, summary (500 chars), url, category
```

### **Источник 2: Hacker News** (`fetch_hackernews_stories`)
```
🔍 Поиск: топовые истории с ключевыми словами
   (ml, ai, security, exploit, vulnerability, crypto, llm, model, privacy, cyber)
⏱️ Фильтр: нет (берём текущий ТОП)
📊 Объём: до 8 историй
💬 Комментарии: топ-4 комментария к 3 самым обсуждаемым
📦 Данные: title, author, score, url, num_comments, top_comments
```

### **Источник 3: Hugging Face** (`fetch_huggingface_trending`)
```
🔍 Поиск: трендовые модели и датасеты (по количеству скачиваний)
⏱️ Фильтр: нет (берём текущий ТОП)
📊 Объём: до 6 моделей + 6 датасетов = 12 items max
📦 Данные: 
   • Модели: modelId, description, downloads, likes, tags, url
   • Датасеты: id, description, downloads, likes, tags, url
```

## 🔄 Как материалы попадают в итоговый пост

```
ШАГИ:
1. Все_материалы (arXiv + HN + HF) → all_materials (List[Dict])

2. JSON сериализация (ensure_ascii=False) → промт в OpenRouter

3. АНАЛИЗ через AI:
   ├─ Отбор 5-8 наиболее значимых
   ├─ Оценка: новизна, актуальность, практическая важность
   ├─ Ранжирование по критериям
   └─ Добавление рейтинга и тегов

4. Форматирование ответа (Markdown + эмодзи):
   ├─ 🔝 ТОП НАХОДОК ДНЯ
   ├─ Для каждого:
   │  ├─ Заголовок (с переводом если нужен)
   │  ├─ Источник + URL
   │  ├─ ⭐ Рейтинг (1-10)
   │  ├─ 🏷️ Теги (#tag1 #tag2 ...)
   │  ├─ Суть (2-3 предложения)
   │  ├─ Почему важно
   │  └─ Реакция сообщества
   └─ 📌 ГЛАВНЫЕ ВЫВОДЫ ДНЯ (3 ключевых тезиса)

5. PODCAST GENERATION:
   ├─ Генерирует сценарий диалога (2 ведущих)
   ├─ Обсуждение ТОП-3 материалов
   ├─ TTS озвучка (Edge или gTTS)
   └─ Сохранение как podcast.mp3

6. TELEGRAM DELIVERY:
   ├─ Отправка текстового дайджеста (с разбиением по 4096 символов)
   ├─ Отправка audio-файла (podcast)
   └─ ✅ Готово!

7. METADATA SAVE:
   └─ digest_YYYYMMDD.json с полной историей
```

## 🎙️ Podcast-версия

Генерируется как диалог:
```
[АЛЕКСЕЙ]: Здравствуй, Мария! Сегодня нас ждут интересные находки...

[МАРИЯ]: Согласна! Начнём с первого материала — новый метод защиты LLM.

[АЛЕКСЕЙ]: Да, это действительно важно. Я бы отметил, что...

[МАРИЯ]: Хороший момент, но стоит добавить, что на практике...

... (диалог про 3 главных материала)

[АЛЕКСЕЙ]: Итак, главные выводы дня...
```

TTS озвучивает этот сценарий в более естественном виде (не просто чтение, а имитация подкаста).

## 🎯 Особенности

✅ **Полностью бесплатно** — все API бесплатные (arXiv, HN, HF, OpenRouter free)  
✅ **Русский язык** — весь анализ и дайджест на русском  
✅ **Структурированный** — рейтинги, теги, красивая разметка
